1. Introduccion

¿Qué es CSS? significa hojas de estilo en cascada (Cascading Style Sheets) describe cómo se deben mostrar los elementos HTML en la pantalla. Puede controlar el diseño de varias páginas web a la vez. Las hojas de estilo externas se almacenan en archivos CSS.

¿Como le decimos al HTML que archivo CSS tiene que usar? Tenemos que agregar lo siguiente al elemento head. 

<link rel="stylesheet" src="styles.css" />

En este caso styles.css es el archivo de css que contendra todos los estilos. Podemos utilizar varios archivos, pero en este ejemplo utilicemos solo uno.

2. Modelo de caja (Box model)

Todo lo que muestra CSS es una caja. Por lo tanto, comprender cómo funciona el modelo de caja CSS es una base fundamental de CSS.

Digamos que tienes este fragmento de HTML: 

<p>
    I am a paragraph of text that has a few words in it.
</p>

Luego escribiendo este CSS para el parrafo anterior: 

p {
    width: 100px;
    height: 50px;
    padding: 20px;
    border: 1px solid;
}

See the Pen Untitled by web.dev (@web-dot-dev) on CodePen.

El contenido se sale de su elemento y tendría 142px de ancho, en lugar de 100px.

¿Porqué sucede eso? 142 = 100 de ancho + 40 (padding izq + der) + 2 (border izq + der).

El modelo de caja es una base fundamental de CSS y comprender cómo funciona, cómo se ve afectado por otros aspectos de CSS y, lo que es más importante, cómo puedes controlarlo te ayudará a escribir un CSS más predecible.

2.1 Contenido y tamaño

Las cajas tienen un comportamiento diferente según su display, sus dimensiones establecidas y el contenido que vive dentro de ellas. Este contenido podría ser incluso más cajas, generadas por elementos secundarios, o por contenido de texto sin formato. De cualquier manera, este contenido afectará la forma predeterminada del tamaño de la caja.

Puedes controlar esto mediante el tamaño extrínseco, o puedes seguir permitiendo que el navegador tome decisiones por ti en función del tamaño del contenido, utilizando el tamaño intrínseco.

En una caja con dimensiones fijas y un borde grueso. La caja tiene un ancho, por lo que tiene un tamaño extrínseco, el cual controla el tamaño de su contenido hijo. Sin embargo, el problema con esto es su contenido es demasiado grande para la caja, por lo que se desborda fuera del borde de la caja del elemento padre.

Una forma de evitar este desbordamiento es permitir que la caja tenga un tamaño intrínseco ya sea removiendo el tamaño del ancho o, en este caso, estableciendo el width en min-content. El min-content le dice a la caja que sea tan ancha como el ancho mínimo intrínseco de su contenido.

Esto permite que la caja se ajuste perfectamente al texto.. 

p {
    width: min-content;
    height: min-content;
    padding: 20px;
    border: 1px solid;
}

2.2 Las areas del modelo caja

Las cajas se componen de áreas de modelo de caja distintas que realizan un trabajo específico.

Inicias con la caja de contenido, que es el área en la que reside el contenido. Como aprendiste antes: este contenido puede controlar el tamaño de su elemento principal, por lo que suele ser el área de tamaño más variable.

La caja de padding (relleno) rodea la caja de contenido y es el espacio creado por la propiedad de padding. Debido a que el padding está dentro de la caja, el fondo de la caja será visible en el espacio que crea.

La caja de borde rodea la caja de padding y su espacio está ocupado por el valor del border. La caja de borde son los límites de su caja y el filo del borde es el límite de lo que puedes ver visualmente. La propiedad de border se usa para enmarcar visualmente un elemento.

El área final, la caja de margen, es el espacio alrededor de tu caja, definida por la regla de margin. Propiedades como el outline y box-shadow ocupan este espacio porque están pintadas por encima de los elementos, por lo que no afectan el tamaño de nuestra caja.

2.3 Una analogia util

El modelo de caja es complejo de entender, así que recapitulemos lo que has aprendido con una analogía.

En este diagrama, tienes tres fotos con marcos, montados en una pared, uno al lado del otro. El diagrama tiene etiquetas que asocian elementos del marco con el modelo de caja.

Para descomponer esta analogía:

La caja de contenido es la obra de arte.
La caja de padding es el blanco mate, entre el marco y la obra de arte.
La caja de borde es el marco, que proporciona un borde literal para la obra de arte.
La caja de margen es el espacio entre cada cuadro.
La sombra ocupa el mismo espacio que la caja de margen.

3. Selectores

Para aplicar CSS a un elemento es necesario seleccionarlo. Imaginemos que queremos que el primer párrafo de un artículo sea un texto de tamaño grande y color rojo. 

<article>
    <p>Esta parte es mas grande y de color rojo.</p>
    <p>Esta parte tiene tamaño normal y el color por defecto.</p>
</article>

Utilizamos un selector CSS para seleccionar ese elemento específico y aplicar reglas de CSS: 

article p:first-of-type {
    color: red;
    font-size: 1.5em;
}

3.1 Partes de una regla CSS

Una regla CSS es un bloque de código, que contiene uno o más selectores y una o más declaraciones.

En esta regla CSS, el selector es .my-css-rule el cual selecciona todos los elementos con la clase my-css-rule. Hay 3 declaraciones dentro de las llaves { }. Una declaración está compuesta por una propiedad y un valor que aplica estilos a los elementos hallados por el selector. Una regla CSS puede tener tantas declaraciones y selectores como se desee.

3.2 Selectores simples

Selector universal: selecciona todos los elementos. 

* {
 color: blue;
}

Esta regla hace que cada elemento HTML de la página tenga texto color azul.

Selector por tipos: selecciona un elemento HTML directamente. 

section {
    padding: 2em;
}

Esta regla hace que todos los elementos <section>. tengan un padding. de 2em. en todas las direcciones.

Selector por clases: Un elemento HTML puede tener una o más clases aplicadas en el atributo class. El selector de clases selecciona cualquier elemento que tenga una clase en particular aplicada. 

<div class="my-class"></div>
<button class="my-class"></button>
<p class="my-class"></p>
.my-class {
 color: red;
}

Todos los elementos que tengan aplicada la clase my-class tendran texto de color rojo. En este caso son los elementos div, button y p

A pesar de tener otras clases aplicadas, el siguiente elemento también tendrá aplicada la clase my-class. 

<div class="my-class another-class some-other-class"></div>

Selector por ID: Si un elemento HTML tiene un atributo id con un cierto valor, este deberia ser el unico elemento con dicho valor. Por ejemplo, si tenemos un div con id=”contenedor”, entonces el unico elemento con id=”contenedor” es ese mismo elemento. 

#contenedor {
 border: 1px solid blue;
}

Este CSS aplica un borde azul al elemento HTML que tiene el id “contenedor”. 

<div id="rad"></div>

Notese que el selector de clases tiene un punto seguido del nombre de la clase .mi-clase, mientras que el selector de id tiene un # seguido del id en cuestion, por ejemplo #mi-id.

Selector por atributos: selecciona todos los elementos que contenga un atributo HTML especifico, los atributos van entre [ ]. 

[data-type='primary'] {
    color: red;
}

Este CSS le aplica texto de color rojo a todos los elementos que tengan un atributo data-type con el valor primary.

Como este: 

<div data-type="primary"></div>

Agrupando selectores: Se pueden utilizar muchos selectores, separándolos con coma. 

strong,
em,
.my-class,
[lang] {
    color: red;
}

Este ejemplo cambia el color a todos los elementos <strong> y <em>. También a los elementos que tengan la clase .my-class. y los que tengan el atributo lang.

3.3 Pseudo-clases

Los elementos HTML pueden tener distintos estados, por ejemplo cuando un elemento tiene el mouse encima se encuentra en estado "hover". Para estas situaciones, se utiliza la pseudoclase :hover. 

a:hover {
 outline: 1px dotted green;
}

En este ejemplo aplica un outline punteado de color verde, a todos los elementos a que se encuentran en estado hover. Es decir, cuando pasamos el mouse por encima.

4. Cascada

CSS son las siglas para Cascading Stylesheets (hojas de estilo en cascada). A veces, dos o más reglas CSS pueden aplicarse a un elemento HTML, el algoritmo de cascada es quien se encarga de resolver los conflictos que esto pueda provocar.

En este ejemplo el texto del botón con el siguiente CSS será azul a pesar de que primero se lo hizo rojo. 

button {
    color: red;
}

button {
    color: blue;
}

El algoritmo de cascada tiene en cuenta el orden en que aparecen las reglas CSS y cómo aparecen mientras calcula la resolución de conflictos. En el ejemplo anterior, hay dos reglas que tienen selectores de especificidad idéntica, por lo que la última en ser declarada fue la que ganó, los elementos se vuelven más específicos cuanto más abajo están declarados en la página.

Los estilos pueden provenir de varias fuentes en una página HTML, por ejemplo de una etiqueta <link>, y una etiqueta <style> incrustada y CSS en línea como se define en el atributo de style. El orden también se aplica a dichos elementos, el que se encuentre declarado más abajo tendrá mayor especificidad.

4.1 Especificidad

La especificidad es un algoritmo que determina qué selector de CSS es el más específico, utilizando un sistema de “puntuación” para realizar esos cálculos. Al hacer una regla más específica que otra, puede hacer que se aplique incluso si algún otro CSS que coincida con el selector aparece más adelante en el CSS.

El CSS dirigido a la clase en un elemento hará que la regla sea más especíhica y se considerará más importante de aplicar que el CSS dirigido solamente al elemento HTML.

Un ID hace que el CSS sea aún más específico, por lo que los estilos aplicados a una ID anularán los aplicados de otras formas.

Un atributo style en línea con CSS declarado anulará todos los demás CSS, independientemente de su posición, a menos que una declaración tenga definido el atributo de !important


Orden de mayor a menor especificidad:

  1. !important.
  2. Estilos en linea.
  3. IDs.
  4. Clases, pseudo-clases, atributos.
  5. Elementos y pseudo-elementos.

La especificidad es acumulativa

Como ya vimos arriba, a cada tipo de selector se le otorgan “puntos” que indican qué tan específico es. Los puntos de todos los selectores que usamos para apuntar a un elemento se suman dando como resultado su especificidad. Esto significa que si apuntamos a un elemento con una lista de selección como a.my-class.another-class[href]:hover, obtenemos algo bastante difícil de sobrescribir con otro CSS, por esta razón es una buena idea mantener los selectores lo más simples posible.

Origen

El CSS que escribimos no es el único CSS que se aplica a una página. El algoritmo de cascada también tiene en cuenta el origen de los CSS. Este origen incluye la hoja de estilo interna del navegador, los estilos agregados por las extensiones del navegador o el sistema operativo y tu CSS creado.

El orden de especificidad de estos orígenes, de mayor a menor, es el siguiente:

  1. Agente de usuario !important: Cualquier !important que se define en el CSS predeterminado, proporcionado por el navegador.
  2. Estilos de usuarios locales !important: Cualquier !important que provenga del nivel del sistema operativo o del nivel de las extensiones del navegador.
  3. Los !important creados: Cualquier !important que agregamos en nuestro CSS.
  4. CSS creado: El resto de nuestro CSS.
  5. Estilos de usuarios locales: Pueden provenir del nivel del sistema operativo, por ejemplo un tamaño de fuente base. También pueden provenir de extensiones del navegador, por ejemplo una extensión de navegador que permite al usuario escribir su propio CSS personalizado para una página web.
  6. Estilos base de agente de usuario: Son los estilos que el navegador aplica a los elementos HTML de forma predeterminada.

Importancia

No todas las reglas de CSS se calculan de la misma manera entre sí, ni se les da la misma especificidad entre sí.

El orden de importancia, de mayor a menor, es el siguiente:

  1. Reglas de tipo transition
  2. Reglas de tipo !important
  3. Reglas de tipo animation
  4. Reglas de tipo normal, cómo font-size , background o color

Los tipos de reglas de transition (transición) y animation (animación) tienen más importancia que las reglas normales. En el caso de las transiciones, es incluso más importante que las reglas de tipo !important . Esto se debe a que cuando una animación o transición se activa, su comportamiento esperado es cambiar el estado visual.
